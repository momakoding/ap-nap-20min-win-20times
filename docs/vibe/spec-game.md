# 元首午休20分钟 AP-NAP: 20 Minutes (Tonald Dump Win-Win Edition)

本策划案专为 3 小时 Vibecoding Jam 设计。在此版本中，主角变更为**金发狂人 Tonald Dump（汤纳德·懂普）**。玩家将站在他的视角，在 APEC 峰会的 20 分钟午休间隙，一边通过其独特的“懂式养生法”恢复健康，一边用极具个人风格的推特修辞与外交手腕，将各方政要、媒体记者的打断转化为一场“赢了又赢”的伟大交易。

---

## 一、 游戏概述与核心机制

玩家扮演主角 **Tonald Dump**。游戏总长 **120 秒**现实时间（对应游戏内 20 分钟，1现实秒=10游戏秒）。共触发 **12 次**打断事件，均匀分布于第 5 秒至第 115 秒之间。

* **健康值 (health)**：初始 40，上限 100。降至 0 时触发”彻底气炸/心脏停摆”结局。
* **外交声誉 (diplomacy)**：初始 60，上限 100。降至 0 时触发”被踢出群聊”结局。
* **压力值 (stress)**：初始 30，上限 100。压力 **> 70** 时每现实秒被动扣 1 点健康；达到 100 时立即触发”摔手机破防”结局。

**决策倒计时**：每次打断事件弹出后玩家有 **8 秒**选择，超时自动选择 C 选项（延后）。主游戏计时在决策期间**暂停**。

**行动被打断**：执行健康行动中途若触发打断事件，按已完成比例给予 **50% × 比例**的部分收益，然后中止行动进入决策界面。

### 2.1 行动重复惩罚机制（Repeat-Penalty，当前实现）

同一行动多次使用时，收益按**衰减系数**递减，但**不会被锁死**——懂普永远可以再吃一个汉堡，只是第 N 个汉堡越来越不香。

**衰减公式：**

```text
effectMultiplier = 1 / (1 + useCount * DECAY_FACTOR)
```

推荐 `DECAY_FACTOR = 0.5`，效果曲线如下：

| 第几次使用 | `useCount`（触发时的已用次数） | 效果倍率 |
| --- | --- | --- |
| 第 1 次 | 0 | 1.00 (100%) |
| 第 2 次 | 1 | 0.67 (67%) |
| 第 3 次 | 2 | 0.50 (50%) |
| 第 4 次 | 3 | 0.40 (40%) |
| 第 5 次+ | n | 趋近于 0，但永不锁死 |

健康收益和压力变动均乘以该倍率（四舍五入取整）。UI 上用图标或文字提示"效果已减弱"，但按钮始终可点。

---

> **📦 存档方案：累计选择硬锁（Hard-Lock，备用）**
>
> 以下为旧版机制，数值设计完善后可替换回来：
>
> * 每个行动最多有效使用 **2 次**；第 2 次使用效果减半（`multiplier = 0.5`）；
> * 第 3 次点击时按钮置灰锁定（`count >= 2` 直接 return），无法再触发。
>
> **代码关键路径（`use-game-state.ts`）：**
>
> ```ts
> const count = actionUseCount.value[action.id] ?? 0
> if (count >= 2) return  // 硬锁：第三次起无效
> currentActionMultiplier = count === 0 ? 1 : 0.5
> actionUseCount.value[action.id] = count + 1
> ```
>
> 此方案惩罚更硬，节奏感更强，但容易让玩家在后期卡手动弹。适合数值经过完整测试后、希望逼玩家轮换行动的版本。
